> 给刘老师的完整部署指南。整个流程约 30 分钟。

## 🚀 二、部署到 Streamlit Cloud

### 步骤 1：上传到 GitHub

1. 在 GitHub 创建一个新 repo（建议名字：`cilang-platform` 或 `huawentong-ci`）
2. 把 `cilang_platform/` 目录里的所有文件传上去
3. 仓库可以是 **private**（不影响 Streamlit Cloud 访问）

### 步骤 2：在 Streamlit Cloud 创建应用

1. 登录 https://share.streamlit.io
2. 点 **New app**
3. 选择刚才的 GitHub repo
4. **Main file path** 填：`app.py`
5. **App URL** 自定义一个好记的（如 `cilang-platform`）
6. 点 **Deploy**

> 等待 2-3 分钟首次部署完成。

### 步骤 3：本次部署不需要任何 Secrets 配置

✅ **完全不调用 API**，所以不需要在 Streamlit Secrets 设置 API key
✅ 老师密码和学生数据都存在 SQLite 文件里

## 🌱 三、首次部署后：初始化数据

部署成功后，您**第一次访问网站**时，需要做一件事：**让 Streamlit 跑一次 `seed_data.py` 把《培养好习惯》预置进去**。

### 方法 A（推荐）：在老师后台手动创建

1. 访问您的 Streamlit 网址
2. 点首页 **「👩‍🏫 老师入口」**
3. 注册一个老师账号（用户名 + 密码 + 显示名）
4. 在侧边栏添加班级（如：1A、2E3）
5. 进入「📚 题库管理」→「➕ 创建课文」→ 填课文信息
6. 进入「📥 导入题库」→ 把 V3 题库的 JSON 粘贴进去

### 方法 B（更快）：用 seed_data.py 一键预置

如果想直接看到《培养好习惯》的演示效果：

1. 在 Streamlit Cloud 应用的 **Manage app → Settings → Secrets** 加入：
   ```
   SEED_ON_BOOT = "true"
   ```
2. 重新部署
3. 系统会自动创建示范账号 `demo / demo123` 和《培养好习惯》课文

> 但更推荐方法 A，您可以亲手走一遍流程，熟悉系统。

## 👨‍🎓 四、学生使用流程

1. 访问 Streamlit 网址（您的子域名）
2. 输入 **班级 + 学号 + 姓名**（每次都要填，学生不需要注册）
3. 选「年级 → 单元 → 课文」
4. 进入闯关页面（V3 体验，含语音、动画）
5. 完成所有关卡 → 看到「词语图鉴」
6. 回到 Streamlit 页面 → 点 **「✅ 我已完成，提交成绩」**
7. 数据存入数据库

> ⚠️ 学生必须点「提交成绩」按钮，否则数据不会进数据库！

## 👩‍🏫 五、老师使用流程

### A. 添加新课文 + 题库（每次新课）

1. 进「📚 题库管理」→「➕ 创建课文」→ 填年级/单元/第几课/标题
2. 打开 `claude_prompt.md`，复制整个 Prompt 到 **Claude.ai 网页版**
3. 在 Prompt 末尾填您的课文内容和词语列表
4. Claude.ai 输出 JSON
5. 回到 Streamlit「📥 导入题库」→ 选刚创建的课文 → 粘贴 JSON → 导入

### B. 查看学生数据

进「📊 老师后台」，三个标签页：

| 标签页 | 看什么 |
|---|---|
| 📈 学生总表 | 每个学生完成了哪些课、正确率多少 |
| 📊 班级错词统计 | 哪些词全班错得最多（决定课堂重点） |
| 📋 详细答题记录 | 每个学生每道题选了什么 |

支持**下载 CSV** 方便您课后分析。

## ⚙️ 六、技术细节（出问题时排查用）

### 数据库

- 文件名：`cilang.db`（Streamlit Cloud 持久化）
- 模式：WAL（提高并发）
- 备份建议：定期下载 `cilang.db` 到本地

### 老师权限设计

- 每个老师**只看自己负责班级**的学生数据
- 班级在「老师后台」的侧边栏添加
- 学生填的班级名必须和老师注册的班级名一致（否则数据进库但老师看不到）

### 学生身份识别

- 用 `(班级 + 学号)` 作为唯一标识（不依赖姓名）
- 学生可以重新登录、重做（系统会保留所有历史记录）

### V3 HTML 嵌入方式

- 用 `streamlit.components.v1.html()` 嵌入
- 题库通过字符串替换注入 HTML
- 答题数据通过 `sessionStorage` 和 `localStorage` 在 HTML/Streamlit 之间传递

## 🧯 七、常见问题排查

### Q1：学生答完点提交，提示「还没有检测到完成数据」

**原因**：HTML 还没把数据写入 localStorage，或学生还没看到「词语图鉴」页面。

**解决**：
- 让学生**先完成所有关卡**（包括最后一个词的所有关），看到彩色花纸+词语图鉴
- 等 1-2 秒后再点「提交成绩」按钮

### Q2：老师后台看不到学生数据

**原因**：学生填的班级名和老师注册的班级名**不一致**。

**解决**：
- 在「📊 老师后台」侧边栏检查您的班级列表
- 让学生重新答题时输入完全相同的班级名
- **建议**：开学时让全班学生用统一的班级名（如全部 `1A`，不要有人填 `1a` 有人填 `1 A`）

### Q3：JSON 导入失败

**原因**：Claude.ai 生成的 JSON 格式问题。

**解决**：
- 系统有 3 层自动修复（去除 markdown 包裹、正则提取）
- 如果还失败，看错误提示
- 最稳妥：让 Claude.ai 重新输出，强调「只输出 JSON，不要任何说明文字」

### Q4：老师密码忘了

**解决**：暂无找回功能（毕竟是小型系统）。可以：
- 删除 `cilang.db` 重新开始（**会丢学生数据**）
- 或者请技术朋友在数据库里直接重置密码

### Q5：怎么从 GitHub 更新代码？

Streamlit Cloud 会**自动检测 GitHub 更新**，您 push 代码后 1-2 分钟就会自动重新部署。
**注意**：重新部署不会删除 `cilang.db`，学生数据不会丢。
