# claude_prompt_reading_game.md — 词语·精读闯关 出题规范 v1

> 用途:生成「词语·精读闯关」(词语闯关第四关)的平台 JSON。
> 触发:当我说"做精读关 / 出精读JSON / 做游戏化阅读"时,按本规范执行。
> 本模块 ≠ 阅读理解(华文通·读)。这里训练**词语层**(语境认词+间隔复习),不考篇章理解。

---

## 一、我会提供什么(输入)

1. **课文题目 + 年级/单元/课次**(如:中一 单元四 第三课《差不多先生》)
2. **本课词语清单**(全部做挖空考点,一个不漏)
3. **复习范围**(可选):要在叙述里"曝光复习"的旧词/成语来源
   - 中一课文 → 复习中一前面单元的词 + 成语1–70
   - 中二课文 → 复习中二前面单元的词 + 成语71–138
   - 若我不指定,按上述默认
4. **文体**(记叙/说明/议论,影响故事写法和卡片genre标签)
5. 特殊要求(如某词的课本义、要避开的干扰项)

## 二、你要产出什么(输出)

一份完整平台信封 JSON,直接可粘贴导入,不要省略、不要示意。
生成后必须自检(见第六节)并报告自检结果。

---

## 三、JSON 信封格式(严格遵守)

```json
{
  "lesson_meta": {
    "format": "reading_game",          ← 固定,路由靠它
    "linked_vocab_id": null,           ← 固定null,老师在平台里做关联
    "stage": "关4·精读",
    "title_cn": "课文标题",
    "grade": "Sec 1",  "unit": "单元四",  "lesson_no": "第三课"
  },
  "reading_game": {
    "chunks": [ ...4–5段,见第四节... ],
    "card":   { ...精灵卡,见第五节... }
  }
}
```

## 四、chunks(故事与挖空)规则

**故事**
- **原创**小故事/小短文,围绕课文主题,不复制课文原文、不复制原著(如胡适原文)
- 切成 **4–5 段**(chunk),每段 60–120 字、含 **1–3 个挖空**,一屏一段(轻交互纪律)
- 每段: `icon`(1个emoji) + `label`(2–6字小标题)
- 段内 `segments` = `{text}` 与挖空对象交替

**挖空(每个本课词语一个,100%全覆盖)**
```json
{ "w": "词语", "options": ["A","B","C"], "correct": 0,
  "tip": "…", "errtag": "…" }
```
- **3 个选项**,`correct` 为正确项下标
- **干扰项必须来自五大偏误库**(英语负迁移/量词/近义混淆/同音别字/成语降维),不许凭空造
- **答案唯一**:若某词有真近义词会造成"两个都对",不用真近义词当干扰,改用形近/别字/反义硬锁;实在锁不死的,在交付说明里标 [MED]/[LOW] 请老师裁
- **答案位置分布**:correct 的 0/1/2 各占 25–40%,不得连续 3 题同位置
- `tip`:两行,第一行讲正确项(🧩拆字/🔗搭配/✨记忆/💡窍门/🇸🇬新加坡注意),第二行讲干扰项为何错(⚠️)。选对选错都显示同一条tip
- `errtag`:错误类型短标签(如"同音别字·磁器"/"英语负迁移·recycle"),入库供诊断

**复习曝光(不考)**
- 把 3–6 个旧词/旧成语自然编进 `text` 叙述里,只曝光、不挖空、不加长篇幅
- 优先选与主题呼应的(如《差不多先生》配 得过且过/半途而废)

## 五、card(精灵卡)规则

```json
"card": {
  "name": "X灵",                      ← 2–3字,从课文核心意象取名(晶瀑灵/一撇灵/丝路灵)
  "types": ["文体", "属性"],           ← 文体∈{叙事,写景,说明,议论}; 属性1字(水/墨/海/火…)
  "set_no": "中一·0NN",               ← 年级+编号
  "abilities": [
    { "n": "招式名(4字)", "d": "一句话招式说明——必须织入本课词语,让看卡=复习" },
    { "n": "…", "d": "…" }            ← 共2条
  ],
  "creature": {
    "body":   "#十六进制主色",         ← 从课文意象取色
    "accent": "#十六进制点缀色",
    "crest":  "glass | brush | boat", ← 只能三选一(头顶装饰)
    "emblem": "gem | seal | compass"  ← 只能三选一(胸口徽章)
  }
}
```
⚠️ **crest/emblem 只能用上面列出的值**,模板不认识别的(写错=渲染不出来)。
新精灵的差异靠 body/accent 配色区分;要新增 crest/emblem 造型需先改模板,单独提出。

## 六、交付前自检(必做并汇报)

1. JSON 可解析(json.loads 通过)
2. 本课词语 100% 全部有挖空,数量 = 词语清单数
3. correct 位置分布 25–40%,无连续3题同位置
4. crest ∈ {glass,brush,boat},emblem ∈ {gem,seal,compass}
5. 每个挖空有 tip(两行)和 errtag
6. 把握度不足的干扰项/搭配,逐条标 [HIGH]/[MED]/[LOW] 请老师裁

## 七、老师端使用路径(备忘)

阅读理解管理 → 创建新课文 → 导入JSON(粘贴) → 发布 → 🔗关联词语闯关。
**不要**贴到「题库管理」(那里只收词语闯关的数组JSON,会报"必须是数组")。
